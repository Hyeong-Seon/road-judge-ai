## 📝 프로젝트 개요 (Project Overview)

**Road Judge AI**는 교통사고 발생 시 복잡하고 오래 걸리는 과실 비율 산정 과정을 AI를 통해 자동화하여 개인에게 1차적인 판단 근거를 제공합니다. 더불어 비사고 주행 영상을 분석하여 운전자의 습관을 점수화하고 개선점을 제안하는 2차 목표를 가지고 있습니다.

### 📅 개발 배경

- **분쟁의 증가**: 과실비율 분쟁 심의 청구 건수 매년 증가 (2019년 10.2만 건 → 2023년 13.6만 건)
- **느린 처리 속도**: 민사 소송 시 최소 4개월에서 최대 3년 이상 소요
- **비용 부담**: 변호사 선임 및 소송 비용 발생

## 💾 데이터셋 구조 (Dataset Structure)

본 프로젝트는 **AI-Hub 교통사고 영상 데이터**를 활용하여 학습을 진행합니다.

```bash

│
└── 01.데이터
    ├── 1.Training
    │   ├── 라벨링데이터
    │   │   ├── TL_차대차_영상_T자형교차로 (.json)
    │   │   └── ...
    │   └── 원천데이터
    │       ├── TL_차대차_영상_T자형교차로 (.mp4)
    │       └── ...
    │
    └── 2.Validation
        ├── 라벨링데이터
        └── 원천데이터
```

## 데이터 전처리 (Data Preprocessing)

프로젝트의 현재 단계는 **비디오 데이터 전처리(원본 → 모델 입력)** 입니다.

- **데이터 선택(필터링)**: CCTV 및 3인칭 블랙박스(BB-3) 영상은 관점 차이로 인한 negative transfer를 막기 위해 제외하고, **1인칭 블랙박스(BB-1)** 영상만 사용해 시점 일관성을 유지했습니다.
- **데이터 클리닝**: 이상치와 결측치를 처리했습니다. 최종 ‘사고 유형’만 비어 있는 경우는 객체 탐지/분류에 유효하므로 유지했고, 상세 결과값이 비어 있는 경우는 제외했습니다.
- **비디오 처리(크롭 & 리사이즈)**: **Center Crop(1080×1080) → 224×224 리사이즈**로 처리했습니다. 객체별 바운딩박스 크롭은 영상마다 비율이 달라 공간적 특징이 흔들릴 수 있어, **일관된 공간 특성**을 위해 균일한 센터 크롭 방식을 채택했습니다.
- **구현 상세(코드 특징)**:
  - Google Colab 환경에서 GPU 드라이버 호환 이슈로 **CPU 최적화** 방식으로 처리했습니다.
  - `ThreadPoolExecutor` + `MAX_WORKERS=8`로 Google Drive 읽기 병목을 완화했습니다.
  - **원자적 저장(Atomic Write)**: `.tmp`로 임시 저장 후 성공 시 rename하여 0-byte 파일을 방지했습니다.
  - **재시작 대응**: 이미 처리된 파일은 자동 스킵해 런타임 종료 후 이어서 처리할 수 있습니다.
  - **오류 처리**: FFmpeg 에러 로그와 타임아웃을 포함해 안정성을 확보했습니다.
  - **I/O 효율화**: Colab 로컬 SSD에서 처리 후 ZIP으로 압축해 Drive로 업로드했습니다.
